# 職務経歴

## 概要

機械学習の継続的な本番運用を支えるMLOps領域で多くの経験を積んできました。また、データ活用を促進するための基盤構築にも一定の知見があります。さらに、エンジニアリングマネージャーとして、エンジニアリング組織全体の改善やチームマネジメントにも積極的に取り組んでいます。

## 株式会社 CoeFont MLOpsエンジニア (2023/06〜現在)

### ML基盤の開発

リサーチチームが提供する音声AIモデルの運用・改善を行なっていました。

- 基盤のインフラ開発
  - Datadogによる外形監視およびインフラ監視の導入
  - Sentry 導入によるアプリケーション監視導入
  - SLIの設計および継続的な改善
  - 特定のシステムの AWS EC2 からAWS ECS への移行
  - セキュリティルールの見直しおよびVPCの移行作業
  - AWSインフラのコスト管理のためのSlack通知Botの開発と定期見直し会の設置
  - AWS WAFを活用したDOS攻撃対策用ファイアウォールの導入
  - AWS Step Functionsを利用した機械学習バッチ基盤のアーキテクチャ選定および構築 [プレスリリース](https://prtimes.jp/main/html/rd/p/000000080.000078329.html)
  - CloudFormationを用いた新規推論アプリケーションのECS環境構築
- アプリケーション開発
  - FastAPIを使用した音声AI推論Webアプリケーションの開発
  - Pytestによるユニットテストおよび統合テストの導入
  - GitHub Actionsを利用したCI/CD環境でのLintチェックおよびデプロイフローの整備

### データ分析基盤の整備・社内データ分析文化の醸成

社内のデータ基盤の開発・運用を行なっていました。

GCP上でのデータ基盤(ELT)を整備し、[DynamoDBとAuroraで作るミニマムなデータ基盤の事例紹介](https://zenn.dev/coefont/articles/25f0f649e0b42b) に内容がまとまっています。

- データインフラ基盤の立ち上げ
  - DynamoDBおよびAuroraからのデータ抽出バッチの開発（Extract処理）
  - DataTransferServiceを活用したデータ転送プロセスの構築（Load処理）
  - データ利用者向けのテーブル設計および最適化（Transform処理）
  - ポリシータグを使用したセンシティブデータのアクセス制御設定
  - Cloud Run上でセルフホスティングされたGoogle Tag Managerの導入
  - Stripeやその他テーブルなど、継続的に追加されるデータソースの対応およびメンテナンス業務
  - Terraformを活用したインフラのコード管理（IaC）
- 社内データ分析文化の醸成
  - 分析チュートリアルおよび分析事例をまとめたNotionの整備
  - 全社KPIの開発組織への浸透を目的としたワークショップの開催
  - ダッシュボード作成に関するワークショップの開催

### エンジニアリングマネージャーとして開発組織の改善

エンジニアリングマネージャーとして開発組織の成果最大化を行なっていました。

約20名のエンジニアを2人のEMとVPoEでマネジメントしていました。

- エンジニアリング組織全体の強化
  - 各メンバーとの1on1ミーティングの実施や目標設定（OKR）を通じた成長支援
  - 組織内のヒアリングやプロダクト戦略を考慮し、エンジニア組織の重点採用ポジションを決定
  - 採用方針に基づくリファラル制度の設計・導入
  - 議論の活性化や知識共有を目的としたオフィス出社日の設定とその定着支援
- MLOpsチームのマネジメント
  - チーム定例会の開催および開発ルールの策定
  - チームバックログの整備と優先順位の決定
  - 各プロジェクトへのメンバーアサインと進捗管理
  - 新規メンバーの面接、入社対応、及び育成計画の策定
- その他
  - CREチームの各種オペレーションの構築と改善活動
  - アナリティクスエンジニアの採用基準の策定
  - バックエンドの負債解消を目的とした人材の確保とプロジェクトの推進

## ピクシブ株式会社 機械学習エンジニア (2021/04〜2024/05)

### 社内用レコメンド APIの立ち上げプロジェクト

ピクシブ社が展開する複数サービスにレコメンドを提供するためのAPIを構築しました。[レコメンド API を構築した話](https://inside.pixiv.blog/2022/08/04/120000)

- PHP によるAPIの作成
  - 各種モックを利用したユニットテストの作成
  - Swagger を利用した API ドキュメント作成や仕様と実装を一致させる仕組みの作成
  - セキュリティ要件に基づいたリクエストのパラメタ暗号化の実装
  - 並列実施可能な AB テスト機構の作成
- API 作成後の他サービスチームとのレコメンド導入のプロジェクト推進
  - コサイン類似度ベースでのレコメンドを作成
  - AB テスト後の効果検証

### MLOps 推論基盤の立ち上げプロジェクト

GCP上で機械学習モデルをリアルタイムで推論するための基盤を構築しました。[GCP の推論基盤を作成した話](https://inside.pixiv.blog/2023/03/02/140000)

- インフラ
  - terraform を使った ほぼ全てのGCP リソースの作成
  - GKE/GCS/Artifact Registory/IAP/VPC network/Managed SSL/BQ など
  - kubernetesのためのkustomize によるマニフェストファイル整備
- CI/CD の整備
  - GitlabCI によるデプロイフローの整備
  - Git Opsの実装
- Python によるマイクロサービス API の構築
  - アプリケーションの部分の作成
  - テスト/lint/docker image build のための CI の設定整備
- PHP のバックエンド API 変更
  - GCP クライアントライブラリ作成
- react, next, vue のフロントエンド変更
  - debounce 処理の実装・GTM 用のコンポーネント変更
- Google Tag Manager の整備
  - 設定変更

- FastAPIを用いたベクトル検索Webアプリケーションの作成
- ほぼ全てのGCPインフラの整備
- POとプロダクトとしてリリースさせる方法の模索

  - Jenkins バッチの Airflow 移行やCloud Logging データの ETL 処理
- インスタンス消し忘れ防止のバッチ
  - GCE インスタンスの消し忘れを slack に通知する仕組みの作成

### 採用・広報業務

採用・広報チームを兼任し、チーム内外での採用サポートを行なっていました

- 採用
  - 機械学習エンジニアの採用業務  
    機械学習エンジニアにおける、アルバイト・新卒・中途・業務委託の採用業務を担当。また、応募要項の作成・面接シートの制定・カジュアル面談・書類選考・1 次面接における判断業務を担当
  - インターン・アルバイトのメンター  
    合計で 4 回の受け入れ、そのうち 1 人は新卒入社予定
  - エンジニアの全職種の採用活動  
    中途採用チームを兼任。全職種の中途採用における 1 次面接の社内エンジニアのアサインを担当  
    採用フローの改善や、採用基準の統一化プロジェクトを推進

- 広報
  - 学生向け機械学習コンペティションのスポンサー  
    外部の技術サークルと連携し、企画の立案と実行　[コンペティションへのスポンサード事例](https://inside.pixiv.blog/2023/10/26/180000)
  - アプリエンジニア・デザイナー向けのイベント手伝い  
    他社との調整・会場設営・配信機材の整備などを担当
