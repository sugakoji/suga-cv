# 職務経歴

## 概要

機械学習モデルをwebサービス上で運用するための全てを担当するMLOps領域での経験を積んでいます。また、データ基盤の整備などにも関わっています。そして、技術面だけでなく、エンジニアリングマネージャーとして、エンジニアリング組織全体の改善やチームマネジメントにも取り組んでいます。

## 株式会社 CoeFont MLOpsエンジニア (2023/06〜現在)

### ML基盤の開発

リサーチチームが提供する音声AIモデルの運用・改善に対して責務がありました。

- 基盤のインフラ開発
  - datadog による外形監視・インフラ監視導入
  - sentry 導入によるアプリケーション監視導入
  - 特定のシステムの AWS EC2 からAWS ECS への移行
  - セキュリティルール修正とVPCの移行
  - AWSインフラのコストのslack通知・定期見直し会の策定
  - AWS WAFを用いたDOS攻撃対策のためのファイアウォール導入
  - AWSの機械学習バッチ基盤のリプレース、それに伴うリリースのPjM業務。[プレスリリース](https://prtimes.jp/main/html/rd/p/000000080.000078329.html)
  - aws step functionsを用いた機械学習バッチ基盤のアーキテクチャ選定・作成
- アプリケーション開発
  - FastAPIを用いた音声AIの推論webアプリケーションの作成
  - pytestを用いたテストの導入
  - CI・CD上のlinter・Cfnなどのデプロイフローを実装

### データ分析基盤の整備・社内データ分析文化の醸成

社内のデータ基盤の開発・運用に責任を持っていました。

GCP上でのデータ基盤(ELT)を整備し、[DynamoDBとAuroraで作るミニマムなデータ基盤の事例紹介](https://zenn.dev/coefont/articles/25f0f649e0b42b) に内容がまとまっています。

- データインフラ基盤の立ち上げ
  - DynamoDBとAuroraからデータを抽出するバッチの作成(Extract処理)
  - DataTransferServiceを利用したデータ転送部分の作成 (Load処理)
  - データ利用者のためのテーブル整備 (Transform処理)
  - Policy tagを用いたセンシティブデータの閲覧制限
  - Cloud Run上で動くホスティング型の Google Tag Manager の導入
  - 継続的に発生するデータソース追加の依頼 (stripeや他テーブル)やメンテナンスなどの対応
- 社内データ分析文化の醸成
  - 全社KPIを開発組織に浸透させるワークショップの開催
  - ダッシュボードの策定・KPIドリブンへの誘導

### エンジニアリングマネージャーとして開発組織の改善

エンジニアリングマネージャーとして開発組織の成果最大化に対して責任を持っていました。

約20名のエンジニアを2人のEMとVPoEでマネジメントしていました。

- 担当範囲
  - エンジニアリング組織全体の改善
    - 各メンバーとの1on1実施・目標設定(OKR)
    - 組織内でのヒアリングやプロダクトの戦略を踏まえた上でのエンジニア組織の注力採用ポジションの決定
    - 採用方針を踏まえたリファラル制度の策定
    - 議論の活発化・知識の伝播を狙ったオフィス出社日の策定とその継続的な実施
  - MLOpsチームのマネジメント
    - チーム定例・開発ルールの整備
    - チームバックログの整備・優先順位度の決定
    - 各プロジェクトへのメンバー配置・監督
    - 経営層とのプロジェクトスケジュール制定や注力領域の意思決定
    - 新規チームメンバーの面接・受け入れ・育成
    - チーム内のプロジェクトの細分化・タスクアサイン・スケジュール管理
  - その他
    - CREチームにおける各種オペレーションの整備・改善
    - アナリティクスエンジニアの採用項目の策定

## ピクシブ株式会社 機械学習エンジニア (2021/04〜2024/05)

### 社内用レコメンド APIの立ち上げプロジェクト

ピクシブ社が展開する複数サービスにレコメンドを提供するためのAPIを構築しました。[レコメンド API を構築した話](https://inside.pixiv.blog/2022/08/04/120000)

- php によるAPIの作成
  - 各種モックを利用したユニットテストの作成
  - Swagger を利用した API ドキュメント作成や仕様と実装を一致させる仕組みの作成
  - セキュリティ要件に基づいたリクエストのパラメタ暗号化の実装
  - 並列実施可能な AB テスト機構の作成
- API 作成後の他サービスチームとのレコメンド導入のプロジェクト推進
  - コサイン類似度ベースでのレコメンドを作成
  - AB テスト後の効果検証

### MLOps 推論基盤の立ち上げプロジェクト

GCP上で機械学習モデルをリアルタイムで推論するための基盤を構築しました。[GCP の推論基盤を作成した話](https://inside.pixiv.blog/2023/03/02/140000)

- サーバーの
  - terraform を使った ほぼ全てのGCP リソースの作成
  - GKE/GCS/Artifact Registory/IAP/VPC network/Managed SSL/BQ など
  - kubernetesのためのkustomize によるマニフェストファイル整備
- CI/CD の整備
  - GitlabCI によるデプロイフロー整備
- python によるマイクロサービス API の構築
  - アプリケーションの部分の作成
  - テスト/lint/docker image build のための CI の設定整備
- php のバックエンド API 変更
  - GCP クライアントライブラリ作成
- react, next, vue のフロントエンド変更
  - debounce 処理の実装・GTM 用のコンポーネント変更
- Google Tag Manager の整備
  - 設定変更

  - Reactを用いた検索画面の実装
  - FastAPIを用いたベクトル検索Webアプリケーションの作成
  - ほぼ全てのGCPインフラの整備
  - POとプロダクトとしてリリースさせる方法の模索

### 採用・広報業務

採用・広報チームを兼任し、チーム内外での採用サポートを行なっていました

- 採用
  - 機械学習エンジニアの採用業務  
    機械学習エンジニアにおける、アルバイト・新卒・中途・業務委託の採用業務を担当。また、応募要項の作成・面接シートの制定・カジュアル面談・書類選考・1 次面接における判断業務を担当
  - インターン・アルバイトのメンター  
    合計で 4 回の受け入れ、そのうち 1 人は新卒入社予定
  - エンジニアの全職種の採用活動  
    中途採用チームを兼任。全職種の中途採用における 1 次面接の社内エンジニアのアサインを担当  
    採用フローの改善や、採用基準の統一化プロジェクトを推進

- 広報
  - 学生向け機械学習コンペティションのスポンサー  
    外部の技術サークルと連携し、企画の立案と実行　[コンペティションへのスポンサード事例](https://inside.pixiv.blog/2023/10/26/180000)
  - アプリエンジニア・デザイナー向けのイベント手伝い  
    他社との調整・会場設営・配信機材の整備などを担当
